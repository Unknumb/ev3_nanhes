Contrato confirmado
-------------------
El backend define en api/schema.py:

PredictRequest:
{
  "features": {
    "CODIGO": numero | null
  },
  "edad_cronologica": numero | null
}

PredictResponse:
{
  "es_longevo": boolean,
  "probabilidad": number,
  "edad_biologica": number,
  "edad_cronologica": number | null,
  "gap": number | null
}

El backend valida solo schema["features"] dentro de features.
Por eso edad_cronologica se envia fuera de features.

Como viene exactamente el schema
--------------------------------
feature_schema.json entrega:
- features: lista de campos del modelo.
- extra_inputs: lista de campos extra que no son features del modelo.

Reglas del payload:
- Los campos numeric se convierten con Number(...).
- Los campos categorical con options numericas se convierten a number.
- Los campos vacios se envian como null.
- Los campos opcionales vacios tambien se envian como null.
- edad_cronologica se lee desde extra_inputs y se envia como propiedad top-level.
- edad_cronologica no se incluye dentro de features.

Respuesta recibida
------------------
No se recibio una respuesta live de /predict en esta ejecucion porque la API local no esta levantada y el entorno no tiene uvicorn instalado.

Verificaciones previas realizadas:
- http://localhost:8000/schema no respondia.
- python -m uvicorn api.main:app --host 127.0.0.1 --port 8000 fallo con "No module named uvicorn".
- .\.venv\Scripts\python.exe -m uvicorn api.main:app --host 127.0.0.1 --port 8000 fallo con "No module named uvicorn".

Se dejo implementado el manejo de respuestas:
- 200/OK: muestra el JSON completo devuelto por /predict en una tarjeta temporal.
- 422: lee detail, intenta asociar errores a campos por codigo y los muestra bajo el campo correspondiente.
- 503: muestra "Modelo no disponible actualmente".
- Otros errores: muestra el detail recibido o el status HTTP.

Estados implementados
---------------------
- submitting: boton deshabilitado y texto "Enviando...".
- success: mensaje "Prediccion recibida correctamente".
- error: mensaje de error visible.

Verificacion frontend
---------------------
Comando ejecutado:
cd web
npm.cmd run build

Resultado:
- Next.js compilo correctamente.
- TypeScript paso correctamente.

Como probarlo
-------------
1. Levantar API:
   pip install -r api/requirements.txt
   uvicorn api.main:app --reload

2. Levantar frontend:
   cd web
   npm install
   npm run dev

   En PowerShell, si npm.ps1 esta bloqueado:
   npm.cmd install
   npm.cmd run dev

3. Abrir:
   http://localhost:3000

4. Completar los campos requeridos del formulario y presionar "Enviar a predict".

Resultado esperado:
- El frontend envia POST ${NEXT_PUBLIC_API_URL}/predict.
- Se muestra el payload enviado.
- Si el modelo esta disponible, se muestra el JSON completo de respuesta.
- Si el modelo no esta disponible, se muestra "Modelo no disponible actualmente".

Confirmacion final
------------------
El contrato del frontend coincide con la API:
- features contiene solo codigos de schema.features.
- edad_cronologica viene de schema.extra_inputs y se envia separado.
- Los opcionales vacios se envian como null.
