Estructura de /aggregates usada
-------------------------------
GET ${NEXT_PUBLIC_API_URL}/aggregates

Estructura esperada por la vista:

{
  "total_predicciones": number,
  "pct_longevos": number | null,
  "edad_biologica_promedio": number | null,
  "gap_promedio": number | null,
  "edad_biologica_distribucion": [
    {
      "min": number,
      "max": number,
      "count": number
    }
  ],
  "ultimas": [
    {
      "created_at": string,
      "es_longevo": boolean,
      "probabilidad": number,
      "edad_biologica": number,
      "gap": number | null
    }
  ]
}

Componentes visuales implementados
----------------------------------
- KPIs ejecutivos:
  - total_predicciones
  - pct_longevos
  - edad_biologica_promedio
  - gap_promedio
- Histograma de edad biológica con Recharts:
  - fuente: edad_biologica_distribucion
  - eje X: rango min-max
  - eje Y: count
- Tabla de últimas predicciones:
  - created_at
  - es_longevo
  - probabilidad
  - edad_biologica
  - gap
- Estados de interfaz:
  - loading
  - success
  - error
  - empty state cuando total_predicciones = 0

Como probar
-----------
1. Levantar la API backend en http://localhost:8000.
2. Confirmar que GET http://localhost:8000/aggregates responde correctamente.
3. Levantar el frontend:

   cd web
   npm.cmd run dev

4. Abrir:

   http://localhost:3000/executive
