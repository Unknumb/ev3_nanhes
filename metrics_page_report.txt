Metrics page report
===================

Fecha: 2026-06-20
Rama: feature/web-frontend

Objetivo implementado
---------------------
Se implemento unicamente una pagina funcional para consumir:

GET ${NEXT_PUBLIC_API_URL}/metrics

No se modifico:
- formulario principal
- integracion /predict
- integracion /explain
- visualizacion principal
- diseno final

URL de la pagina creada
-----------------------
http://localhost:3000/metrics

Archivos creados
----------------
- web/app/metrics/page.tsx
- metrics_page_report.txt

Archivos modificados
--------------------
- web/lib/api.ts

Estructura exacta recibida desde /metrics
-----------------------------------------
Confirmada desde api/main.py.

El backend devuelve un objeto JSON donde:
- cada clave es el nombre de un archivo de reporte
- cada valor es el contenido de ese reporte como string

Forma exacta:
{
  "reporte_clasificacion_2015.txt": "string",
  "reporte_regresion_2015.txt": "string"
}

Si un archivo no existe, el backend devuelve:
"(no disponible)"

Ejemplo esperado:
{
  "reporte_clasificacion_2015.txt": "accuracy: 0.87\nprecision: 0.81\nrecall: 0.76\nf1: 0.78\nroc_auc: 0.90",
  "reporte_regresion_2015.txt": "mae: 4.2\nrmse: 5.8\nr2: 0.62"
}

Respuesta real live
-------------------
No se pudo obtener una respuesta live de /metrics en esta ejecucion porque la API local no esta levantada y el entorno no tiene uvicorn instalado.

Implementacion frontend
-----------------------
- web/lib/api.ts ahora exporta METRICS_URL.
- web/app/metrics/page.tsx:
  - carga GET /metrics en el navegador.
  - muestra estado "loading metrics".
  - muestra estado "metrics success".
  - muestra estado "metrics error".
  - si la API no esta disponible, muestra un mensaje amigable con la URL esperada.
  - intenta extraer metricas numericas conocidas desde el texto:
    - accuracy
    - precision
    - recall
    - f1
    - f1-score
    - roc_auc
    - auc
    - mae
    - mse
    - rmse
    - r2
  - muestra tambien el texto crudo completo de cada reporte.

Verificacion frontend
---------------------
Comando ejecutado:
cd web
npm.cmd run build

Resultado:
- Next.js compilo correctamente.
- TypeScript paso correctamente.
- La ruta /metrics aparece en el build.

Como probarla
-------------
1. Levantar API:
   pip install -r api/requirements.txt
   uvicorn api.main:app --reload

2. Levantar frontend:
   cd web
   npm install
   npm run dev

   En PowerShell, si npm.ps1 esta bloqueado:
   npm.cmd install
   npm.cmd run dev

3. Abrir:
   http://localhost:3000/metrics

Resultado esperado
------------------
- Primero se muestra "loading metrics".
- Si la API responde, se muestra "metrics success".
- Se listan los reportes devueltos por /metrics.
- Si se detectan metricas como accuracy, precision, recall, f1, roc_auc o mae, se muestran como tarjetas.
- Siempre se muestra el texto crudo del reporte.
- Si la API no responde, se muestra un mensaje amigable sin romper la pagina.

Confirmacion final
------------------
La pagina /metrics consume unicamente GET /metrics y no modifica ni afecta la UI principal del formulario.
